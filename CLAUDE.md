# AI 協作規範與工作流指南 (CLAUDE.md)

本檔案定義了 AI 助理在與使用者協作時必須嚴格遵守的運作規則與開發標準。本規範旨在確保上下文精簡、提升判斷精準度，並在關鍵節點引入「人機確認閘」。

---

## 🟢 核心指令：開工 (Start Work / Startup)

當使用者輸入關鍵字「**開工**」或「**我來了**」時，AI 助理必須執行以下標準作業程序（SOP）：

1. **環境同步**：
   - 執行 `git pull` 以確保本地代碼與待辦清單與遠端同步。
2. **讀取待辦清單**：
   - 讀取全域 `PENDING.md` 以掌握整體專案狀態。
   - 讀取當前專案目錄下的 `PENDING.md`（專案待辦檔）。
3. **建立今日行動大綱**：
   - 僅列出專案待辦檔中分類為「立即 (Immediate)」與「本週 (This Week)」的待辦任務。
   - 列出今日的第一步行動建議。
4. **【重要】人機確認閘 (Confirmation Gate)**：
   - **限制**：在列出今日行動大綱後，AI 助理必須停止動作，並詢問使用者是否同意開始執行。
   - **禁止行為**：在使用者回覆「同意」、「確認」或給出具體開發指示前，**嚴禁呼叫任何程式碼修改、檔案寫入或自動化腳本執行工具**。

---

## 🔴 核心指令：收工 (End Work / Wrap-up)

當使用者輸入關鍵字「**收工**」或「**下班了**」時，AI 助理必須執行以下標準作業程序（SOP）：

1. **安全掃描**：
   - 掃描變更的代碼檔案，防範 API Key、密碼或敏感憑證外洩。
2. **整理專案待辦**：
   - 檢視今日對話中完成的任務。
   - 將已完成任務從專案 `PENDING.md` 中移除（完成紀錄應保留在 Git 歷史中，不佔用活躍上下文）。
   - 更新專案 `PENDING.md` 的待辦事項與「最後更新時間」。
3. **更新全域摘要**：
   - 執行自動同步腳本（如 `python scripts/sync_pending.py`），自動重算當前專案的待辦個數，並更新全域 `PENDING.md` 的摘要行與時間戳記。
4. **Git 提交與推送**：
   - 執行 `git status` 與 `git diff` 供使用者預覽。
   - 自動生成符合 Conventional Commits 規範的 Commit 訊息。
   - 執行 `git add .`、`git commit` 並將變更推送至遠端倉庫（`git push`）。
5. **對話總結**：
   - 在對話結束前，簡短列出今日已完成事項，並說明下次開工的預期起點。

## 🛠️ 開發與行為準則

1. **語系偏好**：
   - 預設對話與所有文件輸出語系皆為 **繁體中文 (Traditional Chinese)**。
2. **上下文管理**：
   - 保持專案 `PENDING.md` 精簡（建議不超過 100 行），過時或已完成的細節應立即清理。
   - 深度背景資訊（如技術架構、接口設計）應存放在專屬設計文檔中，非必要不強制載入。
3. **程式碼修改原則**：
   - 修改前必須確保理解需求字典與架構規範。
   - 優先尋找既有功能進行重用，禁止未經確認擅自重構或引入不必要的依賴。
